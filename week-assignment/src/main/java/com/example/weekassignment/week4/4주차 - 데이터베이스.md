# 4주차 - 데이터베이스

## 데이터베이스

- 📕 [기본] 데이터베이스가 무엇인지 알아보기

데이터베이스는 구조화된 정보 또는 데이터의 조직화된 모음으로서 일반적으로 컴퓨터 시스템에 전자적으로 저장되며 일반적으로 [데이터베이스 관리 시스템(DBMS)](https://www.oracle.com/kr/database/what-is-database/#WhatIsDBMS)에 의해 제어된다. 연결된 애플리케이션과 함께 데이터와 DBMS를 하나로 묶어 데이터베이스 시스템이라고 하며 단축하여 데이터베이스라고도 한다.

구성요소 : 개체, 속성, 관계

- 📕 [기본] 데이터베이스 자료형 알아보기

데이터베이스에서 사용되는 자료형(Data Types)은 저장할 데이터의 종류를 정의한다. 주로 사용되는 자료형은 다음과 같다(MySQL) :

- **문자열(String)**
    - **`CHAR(n)`**: 고정 길이 문자열 - 타입 크기만큼 데이터가 들어오지 않으면 공백으
    - **`VARCHAR(n)`**: 가변 길이 문자열
    - **`TEXT`**: 매우 긴 문자열
    
    ![Untitled](4%E1%84%8C%E1%85%AE%E1%84%8E%E1%85%A1%20-%20%E1%84%83%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%90%E1%85%A5%E1%84%87%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%89%E1%85%B3%20051e77fd66c44e039a48f4ebd45cee23/Untitled.png)
    
- **숫자(Number)**
    - **`INT`**: 정수형
    - BIGINT
    - **`FLOAT`**: 부동 소수점
    - **`DOUBLE`**: 배정밀도 부동 소수점
    - NUMBER(오라클)
    
    ![Untitled](4%E1%84%8C%E1%85%AE%E1%84%8E%E1%85%A1%20-%20%E1%84%83%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%90%E1%85%A5%E1%84%87%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%89%E1%85%B3%20051e77fd66c44e039a48f4ebd45cee23/Untitled%201.png)
    
- **날짜 및 시간(Date and Time)**
    - **`DATE`**: 날짜 (YYYY-MM-DD)
    - **`TIME`**: 시간 (HH:MM:SS)
    - **`DATETIME`**: 날짜와 시간 (YYYY-MM-DD HH:MM:SS)
    - **`TIMESTAMP`**: 유닉스 타임스탬프
    
    ![Untitled](4%E1%84%8C%E1%85%AE%E1%84%8E%E1%85%A1%20-%20%E1%84%83%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%90%E1%85%A5%E1%84%87%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%89%E1%85%B3%20051e77fd66c44e039a48f4ebd45cee23/Untitled%202.png)
    
- **기타(Other)**
    - **`BOOLEAN`**: 참/거짓 값
    - **`BLOB`**: 바이너리 대형 객체 (이미지, 파일 등)
    
- 📕 [기본] 관계형 데이터베이스가 무엇인지 알아보기

**RDB (Relation Database, 관계형 데이터베이스)**란 키(Key)와 값(Value)들의 관계를 테이블화 시킨 데이터베이스를 의미. 각 테이블은 행(Row)과 열(Column)로 구성되며, 테이블 간의 관계를 통해 데이터를 연결하고 조작할 수 있다. 관계형 데이터베이스는 SQL(Structured Query Language)를 사용하여 데이터를 관리한다.

![Untitled](4%E1%84%8C%E1%85%AE%E1%84%8E%E1%85%A1%20-%20%E1%84%83%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%90%E1%85%A5%E1%84%87%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%89%E1%85%B3%20051e77fd66c44e039a48f4ebd45cee23/Untitled%203.png)

주요 특징:

- **스키마 기반**: 데이터의 구조가 명확히 정의됨
- **테이블 간의 관계**: 외래 키(Foreign Key)를 통해 테이블 간의 관계를 설정
- **트랜잭션**(DBMS에서 하나의 작업의 단위) **지원**: ACID (Atomicity-원자성, Consistency-일관성, Isolation-격리성, Durability-지속) 특성을 지원므로 오류, 실패, 기타 잠재적 오작동에 관계없이 데이터 유효성을 검사할 수 있습니다.

- 📕 [기본] 비관계형 데이터베이스가 무엇인지 알아보기

**비관계형 데이터베이스 (Non-Relational Database)**, 또는 NoSQL 데이터베이스는 전통적인 테이블 구조를 따르지 않는 데이터베이스. 대신, 다양한 데이터 모델을 사용하여 데이터를 저장. 이러한 데이터베이스는 큰 데이터 볼륨, 짧은 지연 시간과 유연한 데이터 모델이 필요한 애플리케이션에 특히 최적화되었으며, 이는 다른 데이터베이스의 데이터 일관성 제한 일부를 완화함으로써 이루어.

주요 유형:

- **문서 지향형(Document-Oriented)**: JSON, BSON 형식으로 데이터를 저장 (예: MongoDB)
- **키-값 저장소(Key-Value Store)**: 키-값 쌍으로 데이터를 저장 (예: Redis)

![Untitled](4%E1%84%8C%E1%85%AE%E1%84%8E%E1%85%A1%20-%20%E1%84%83%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%90%E1%85%A5%E1%84%87%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%89%E1%85%B3%20051e77fd66c44e039a48f4ebd45cee23/Untitled%204.png)

- **컬럼 지향형(Column-Family Store)**: 컬럼 패밀리 형태로 데이터를 저장 (예: Cassandra)
- **그래프 데이터베이스(Graph Database)**: 그래프 구조로 데이터를 저장 (예: Neo4j)

![Untitled](4%E1%84%8C%E1%85%AE%E1%84%8E%E1%85%A1%20-%20%E1%84%83%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%90%E1%85%A5%E1%84%87%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%89%E1%85%B3%20051e77fd66c44e039a48f4ebd45cee23/Untitled%205.png)

- 📕 [기본] 관계형 vs 비관계형 데이터베이스

**관계형 데이터베이스 (RDB)**

- **장점**:
    - 명확한 스키마와 데이터 무결성 보장
    - 복잡한 쿼리와 조인 연산을 효과적으로 처리
    - 트랜잭션 관리가 우수함
- **단점**:
    - 기존에 작성된 스키마를 수정하기가 어려움
    - 대규모 데이터 처리와 수평적 확장(Scale-out)이 어려움

**비관계형 데이터베이스 (NoSQL)**

- **장점**:
    - 다양한 데이터 구조를 저장 가능
    - 수평적 확장이 용이하여 대규모 데이터 처리에 적합
    - 특정 작업(읽기, 쓰기 등)에 최적화된 성능 제공
- **단점**:
    - 데이터 무결성과 트랜잭션 관리가 제한적일 수 있음
    - 복잡한 쿼리와 조인 연산이 어려움
    - 표준화된 쿼리 언어가 없을 수 있음

두 데이터베이스 유형은 각기 다른 사용 사례에 맞춰 선택됨. 관계형 데이터베이스는 데이터 무결성, 복잡한 쿼리, 트랜잭션 관리가 중요한 시스템에 적합하며, 비관계형 데이터베이스는 유연한 스키마, 대규모 데이터 처리, 빠른 읽기/쓰기 성능이 요구되는 시스템에 적합. 

관계형 및 비관계형 데이터베이스(NoSQL 데이터베이스)의 주요 차이점은 데이터를 저장하고 구성하는 방법입니다. 비관계형 데이터베이스는 규칙 기반의 테이블 형식 방식으로 데이터를 저장하지 않는 대신 데이터를 연결되지 않은 개별 파일로 저장하며, 문서 또는 리치 미디어 파일과 같은 복잡하고 구조화되지 않은 데이터 유형에 사용할 수 있음.

관계형 데이터베이스와 달리 NoSQL 데이터베이스는 유연한 데이터 모델을 따르므로 자주 변경되는 데이터를 저장하거나 다양한 유형의 데이터를 처리하는 애플리케이션에 적합.

| 카테고리 | 관계형 데이터베이스 | 비관계형 데이터베이스 |
| --- | --- | --- |
| 데이터 모델 | 테이블 형식. | 키-값, 문서 또는 그래프 |
| 데이터 유형 | 정형. | 정형, 반정형 및 비정형. |
| 데이터 무결성 | 높은 수준으로 전체 ACID 규정 준수. | 최종 정합성 모델. |
| 성능 | 서버에 더 많은 리소스를 추가하는 방식으로 향상됨. | 더 많은 서버 노드를 추가하는 방식으로 향상됨. |
| 확장 | 수평적 규모 조정에는 추가적인 데이터 관리 전략 필요. | 간단하게 수평적 규모 조정 가능. |

## 관계형 데이터베이스(RDB)

- 📕 [기본] 엔티티(Entity)에 대해 알아보기

엔티티(Entity)는 데이터베이스에서 관리하고자 하는 실체나 객체를 의미한다. ex) 학생 정보 시스템 → 학생(Student), 강의(Course), 교수(Professor) 등. 엔티티는 현실 세계의 개체를 데이터베이스에 표현한 것으로, 각 엔티티는 여러 속성(Attribute)을 가질 수 있음.

- 📕 [기본] 테이블(Table)에 대해 알아보기
    
    데이터베이스에서 엔티티를 표현하는 구조로, 행(Row)과 열(Column)로 이루어져 있다.
    
    ![Untitled](4%E1%84%8C%E1%85%AE%E1%84%8E%E1%85%A1%20-%20%E1%84%83%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%90%E1%85%A5%E1%84%87%E1%85%A6%E1%84%8B%E1%85%B5%E1%84%89%E1%85%B3%20051e77fd66c44e039a48f4ebd45cee23/Untitled%206.png)
    
    - 튜플(행: Row)가 무엇인지 알아보기
    - 어트리뷰트(열: Column)가 무엇인지 알아보기
    
- 📕 [기본] 기본키(PK)에 대해 알아보기
    
    기본키(Primary Key, PK)는 테이블 내에서 각 튜플을 고유하게 식별할 수 있는 하나 이상의 속성(열)을 의미합니다. 기본키는 중복된 값을 가질 수 없으며, NULL 값을 허용하지 않다. 기본키는 테이블의 각 행을 고유하게 식별할 수 있게 함으로써 데이터 무결성을 유지하는 데 중요한 역할을 함.
    
    - 단일 테이블만 고려하시면 돼요.
    
    기본키는 단일 테이블 내에서만 유일성을 보장한다. ex) 학생 테이블에서는 학번(ID) 열을 기본키로 설정하여 각 학생을 고유하게 식별할 수 있음.
    
    - 외래키는 아직 학습하지 않으셔도 됩니다.

## 데이터베이스 명령어

- 📕 [기본] DDL 이 무엇인지 알아보기
    
    DDL(Data Definition Language)은 데이터베이스 구조를 정의하거나 변경하기 위해 사용되는 SQL 명령어다.
    
    - `CREATE` 질의어 이해하기
    
    **`CREATE`** 명령어는 데이터베이스에 새로운 객체를 생성하는 데 사용됩니다. 주로 테이블, 뷰, 인덱스 등을 생성하는 데에 활용됩니다. 예를 들어, 새로운 테이블을 생성할 때 사용됩니다.
    
    - `DROP` 질의어 이해하기
    
    **`DROP`** 명령어는 데이터베이스에서 객체를 삭제하는 데 사용된다. 주로 테이블, 뷰, 인덱스 등을 삭제하는 데에 활용된다. 예를 들어, 특정 테이블을 삭제할 때 사용.
    
- 📕 [기본] DML 이 무엇인지 알아보기
    
    DML(Data Manipulation Language)은 데이터를 조작하고 검색하는 데 사용되는 SQL 명령어. 주요 DML 명령어로는 **`SELECT`**, **`INSERT`**, **`UPDATE`**, **`DELETE`**가 있다.
    
    - `SELECT` 질의어 이해하기
    
    **`SELECT`** 명령어는 데이터베이스에서 데이터를 검색할 때 사용. 특정 조건을 만족하는 행을 선택하고 결과를 반환.
    
    - `INSERT` 질의어 이해하기
    
    **`INSERT`** 명령어는 데이터베이스에 새로운 행을 추가할 때 사용. 지정한 테이블에 새로운 데이터를 삽입.
    
    - `UPDATE` 질의어 이해하기
    
    **`UPDATE`** 명령어는 데이터베이스의 특정 행을 업데이트할 때 사용. 특정 조건을 만족하는 행의 값을 변경.
    
    - `DELETE` 질의어 이해하기
    
    **`DELETE`** 명령어는 데이터베이스에서 특정 행을 삭제할 때 사용. 특정 조건을 만족하는 행을 삭제.
    
- 📕 [기본] DCL 이 무엇인지 진~~짜 가볍게 알아보기

DCL(Data Control Language)은 데이터베이스에 대한 접근 권한을 제어하는 데 사용됨. 주요 DCL 명령어로는 **`GRANT`**와 **`REVOKE`**가 있음. 

- 📕 [기본] `COMMIT`, `ROLLBACK` 이 무엇인지 진~~짜 가볍게 알아보기

**`COMMIT`** 명령어는 트랜잭션의 변경 사항을 데이터베이스에 영구적으로 저장하는 데 사용됨. 반대로, **`ROLLBACK`** 명령어는 트랜잭션의 변경 사항을 취소하고 이전 상태로 되돌리는 데 사용됨. 트랜잭션은 데이터베이스에서 일련의 작업을 수행할 때 사용되며, 이를 통해 데이터의 일관성과 무결성을 유지할 수 있음.
